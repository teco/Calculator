Glossário

Salesforce Data Cloud Billable Activities
Este documento fornece um glossário abrangente de atividades no Salesforce Data Cloud que geram cobrança.

Data Service Credits

Data Service Credits são os créditos primários usados para a maioria das tarefas de processamento de dados, harmonização e análise dentro do ecossistema do Data Cloud.

(External) Data Pipeline (Batch & Streaming)

Descrição:
Refere-se à ingestão de dados de qualquer fonte não-Salesforce, como Amazon S3, data lakes externos ou via API. A ingestão de streaming é para dados quase em tempo real de fontes como o conector de Site/Aplicativo Móvel ou a API de ingestão.

Exemplo Prático:
Ingerir 1 milhão de registros de perfil de cliente de um sistema externo via um pipeline em lote consome 2.000 créditos. Ingerir 2 milhões de eventos de interação de carrinho via um pipeline de streaming consome 10.000 créditos.

Consideração:
O processamento em lote é significativamente mais barato que o streaming. Use streaming apenas se o seu caso de uso tiver uma forte justificativa para dados quase em tempo real.

Internal Data Pipeline (Batch & Streaming)

Descrição:
É para a ingestão de dados estruturados de conectores nativos da Salesforce como Salesforce CRM, Marketing Cloud e Commerce Cloud. Esta atividade está incluída e não consome créditos.

Exemplo Prático:
Ingerir 5 milhões de registros de contato do Sales Cloud para o Data Cloud consome 0 créditos.

Consideração:
Embora a ingestão seja gratuita, esses dados ainda consumirão créditos em todos os processos posteriores como transformações, unificação e insights calculados.

Data Transforms (Batch & Streaming)

Descrição:
Envolve a limpeza, padronização ou enriquecimento de dados usando a funcionalidade de transformação do Data Cloud. As transformações em lote são para processamento agendado, enquanto as transformações de transmissão operam em dados quase em tempo real.

Exemplo Prático:
Aplicar uma transformação em lote a 2 milhões de linhas para padronizar códigos de país custaria (2M/1M) * 400 = 800 créditos.

Consideração:
O uso de transformação em lote é calculado com base no número maior de linhas lidas ou escritas. Após a primeira execução, as transformações incrementais processam apenas linhas novas ou alteradas, o que pode reduzir significativamente o uso de créditos.

Profile Unification (Batch)

Descrição:
É o processo de correspondência e fusão de dados de diferentes fontes para criar um perfil de cliente unificado usando conjuntos de regras de resolução de identidade. É uma das atividades que mais consome créditos.

Exemplo Prático:
Unificar 1 milhão de perfis de origem pela primeira vez consome 100.000 créditos.

Consideração:
O consumo é baseado no número de perfis de origem processados, não no número final de perfis unificados. Um perfil de origem inclui um indivíduo e todos os seus registros relacionados (como pontos de contato) incluídos no conjunto de regras. Após a execução inicial, apenas perfis novos ou modificados são processados, mas a alteração de um conjunto de regras aciona uma atualização completa.

Calculated Insights (Batch & Streaming)

Descrição:
Envolve a criação de métricas e dimensões pré-calculadas em seus dados, como o Valor Vitalício (LTV) ou pontuações de engajamento do cliente.

Exemplo Prático:
Um Insight Calculado em lote que une um DMO com 2 milhões de registros e outro com 1 milhão de registros processará 3 milhões de registros no total, custando (3M/1M) * 15 = 45 créditos cada vez que for executado.

Consideração:
O uso é baseado no número de registros em todos os objetos subjacentes usados para criar o insight, não na saída final. Se este insight for usado em um segmento que atualiza diariamente, ele consumirá créditos diariamente.

Data Queries

Descrição:
Abrange qualquer consulta SQL executada contra dados do Data Cloud, como por meio do Editor de Consultas, APIs ou de ferramentas conectadas como Tableau ou CRM Analytics. É uma das operações mais baratas.

Exemplo Prático:
Executar uma consulta que processa 2,5 milhões de registros consumiria (2,5M/1M) * 2 = 5 créditos.

Consideração:
O número de registros processados depende da estrutura da consulta e do total de registros nos objetos sendo consultados. Otimize consultas com cláusulas WHERE para reduzir o custo.

Unstructured Data Processed

Descrição:
É para o processamento de arquivos como PDFs, áudio, vídeo ou documentos de texto, frequentemente para uso com IA e pesquisa vetorial.

Exemplo Prático:
Processar 100 documentos PDF de 1 MB cada seria calculado como 100 MB de uso, consumindo 100 * 60 = 6.000 créditos.

Consideração:
O uso é computado apenas uma vez tanto na "fragmentação" (dividir o arquivo em pedaços) quanto na "vetorização" (criar embeddings para pesquisa de IA).

Data Federation ou Sharing Rows Accessed (Zero-Copy)

Descrição:
Aplica-se ao usar conectores "Bring Your Own Lake" (BYOL) para acessar dados em plataformas externas como Snowflake ou Google BigQuery sem copiá-los para o Data Cloud.

Exemplo Prático:
Consultar uma visualização no Snowflake via federação de dados que retorna 5 milhões de registros consumiria (5M/1M) * 70 = 350 créditos.

Consideração:
Você é cobrado pelo número de linhas recuperadas da fonte cada vez que são acessadas. No entanto, compartilhar dados para a mesma nuvem e região (por exemplo, tanto o Data Cloud quanto o Snowflake estão em AWS us-east-1) não tem consumo de crédito.

Sub-second Real-Time Events & API

Descrição:
É para o processamento de eventos e chamadas de API na camada de sub-segundo, tipicamente usado para personalização em tempo real via os SDKs Web e Móvel. Este é um recurso de alto desempenho que requer uma licença adicional.

Exemplo Prático:
Ingerir 2 milhões de eventos de interação diariamente para personalização em tempo real consumiria aproximadamente 10.000 créditos por dia.

Consideração:
O uso é calculado como a soma de todos os eventos de perfil, eventos de engajamento, chamadas de API e ações de dados processados na camada de tempo real.

Inferences

Descrição:
Uma inferência é uma única saída de um modelo de IA preditivo, seja um modelo nativo do Einstein Studio ou um "Bring Your Own Model" (BYOM) externo.

Exemplo Prático:
Executar um trabalho de previsão em lote em 100.000 registros gera 100.000 inferências e consome (100k/1M) * 3.500 = 350 créditos. Uma única geração de texto de um LLM conta como uma inferência.

Créditos de Segmentação e Ativação

Esses créditos são tipicamente incluídos com a licença do Data Cloud for Marketing ou comprados como um complemento separado. Eles são consumidos quando você cria e ativa públicos.

Segment Rows Processed

Descrição:
Este é o custo de crédito para criar ou atualizar um segmento. Conta o número total de registros de todos os Objetos de Modelo de Dados (DMOs) usados nos critérios do segmento.

Exemplo Prático:
Você constrói um segmento de "Clientes de Alto Valor" filtrando um DMO Individual (5M de linhas) e um DMO de Pedidos relacionado (20M de linhas). O processo contará 25M de linhas, mesmo que o tamanho final do segmento seja de apenas 50.000 pessoas. Isso custaria (25M/1M) * 20 = 500 créditos por atualização.

Consideração:
Considere usar segmentos aninhados para reutilizar públicos existentes ou Insights Calculados para pré-filtrar dados, o que pode reduzir o número de linhas processadas durante a segmentação.

Batch Activation

Descrição:
Este é o custo de publicar um segmento para um alvo de ativação, como o Marketing Cloud Engagement ou um bucket S3.

Exemplo Prático:
Ativar um segmento com 2 milhões de membros consumiria (2M/1M) * 10 = 20 créditos.

Consideração:
Usar atualizações incrementais para ativação é uma estratégia chave para economizar créditos, pois garante que apenas registros novos ou atualizados sejam processados após a execução inicial.

Activate DMO - Streaming

Descrição:
É para ativações quase em tempo real, como enviar um evento para a API de Conversão do Meta assim que um usuário se qualifica para um segmento.

Exemplo Prático:
Uma ativação de transmissão processa 1 milhão de registros novos ou atualizados no DMO. Isso consumiria 1.600 créditos.

Consideração:
Se a ativação usar um grafo de dados, o número de registros processados é duplicado, o que também duplica o custo de crédito.

Outros Add Ons que geram cobrança (Custo Fixo)

Além do consumo baseado em crédito, alguns recursos são precificados como complementos de taxa fixa que você deve considerar em seu custo total de propriedade.

Data Storage

Descrição:
As licenças do Data Cloud vêm com uma alocação de armazenamento base (por exemplo, 1 TB ou 5 TB). Se você exceder isso, deve comprar armazenamento adicional, tipicamente precificado por terabyte por ano.

Data Spaces

Descrição:
Sua organização vem com um espaço de dados padrão. Se você precisar particionar dados para diferentes marcas, regiões ou unidades de negócio, pode comprar Espaços de Dados adicionais por uma taxa anual.

Ad Audiences

Descrição:
Para ativar segmentos para plataformas de publicidade como Google Ads ou Meta, você deve comprar o complemento Ad Audiences, que é tipicamente precificado por público por ano.
